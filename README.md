# 💻 기술 스택 (Tech Stack)
| 분야 | 핵심 기술 | 사용 목적 |
| :--- | :--- | :--- |
| **프론트엔드** | **React** | 사용자 친화적인 UI/UX 및 스케줄링 UI 구현. |
| **백엔드/서버** | **Spring Boot** | **복잡한 매칭 알고리즘 및 예약 트랜잭션** 관리. |
| **보안** | **Spring Security** | 멘토/멘티/관리자 **역할 기반의 인증 및 권한 관리**. |
| **데이터베이스** | **MariaDB** | 멘토/멘티 정보, 예약 기록 등 데이터 저장 및 관리. |

# ⚙️ 핵심 구현 기능 표
| 구분 | 상세 기능 |
| :--- | :--- |
| **지능형 매칭** | 멘토의 전문 분야와 멘티의 요청사항을 종합한 **추천 알고리즘** 구현. |
| **실시간 예약 시스템** | Spring Boot에서 **예약 중복을 방지**하는 견고한 트랜잭션 로직 처리. |
| **역할 기반 API 통제** | Spring Security를 활용하여 역할에 따른 **API 접근 권한 통제** 및 분리. |
| **피드백 시스템** | 멘토링 세션 종료 후 상호 피드백 및 평점을 기록하는 기능 구현. |

# 커밋 메시지

| Type | 목적 | 상세 설명 |
| :--- | :--- | :--- |
| **feat** | **새 기능 추가** | 새로운 기능(Feature)을 코드에 추가했을 때. |
| **fix** | **버그 수정** | 잘못된 코드를 고치거나 버그(Bug)를 수정했을 때. |
| **refactor** | **코드 구조 개선** | 기능 변경 없이 코드 구조를 개선(Refactoring)했을 때. |
| **docs** | **문서 수정** | 코드 외 문서(README, 주석 등)를 수정했을 때. |
| **chore** | **기타 설정 변경** | 빌드 환경, 라이브러리, 환경 설정 등 주요 로직 외 변경. |
| **style** | **스타일/포맷팅** | 코드 로직 변경 없이 스타일, 공백, 포맷팅만 변경했을 때. |
| **test** | **테스트 코드** | 테스트 코드를 추가하거나 기존 테스트를 수정했을 때. |
| **perf** | **성능 개선** | 코드 변경으로 성능(Performance)이 향상되었을 때. |

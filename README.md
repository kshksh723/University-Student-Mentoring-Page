# 👨‍🏫 프로젝트: 대학생 멘토링 페이지 (Mentoring Page)
Figma 디자인 기반, Next.js와 Spring Boot로 구현한 지능형 멘토-멘티 매칭 플랫폼

---

# ✨ 프로젝트 소개 및 디자인 강조
본 프로젝트는 대학생 멘토와 멘티를 연결하는 플랫폼으로, Figma로 완성된 $\text{UX/UI}$ 디자인을 기반으로 개발되었습니다. 
Next.js를 사용하여 $\text{SPA}$ 기반의 사용자 친화적인 $\text{UI}$를 구현하고, Spring Boot로 복잡한 매칭 알고리즘 및 예약 로직의 안정성을 확보합니다.


| 항목 | 내용 |
| :--- | :--- |
| **프로젝트 목표** | 대학생 멘토-멘티를 연결하고, 세션 예약 및 관리 기능을 제공하는 플랫폼 구축. |
| **핵심 아키텍처** | **Spring MVC** 패턴 기반 (JSP를 View 계층으로 통합). |
| **디자인 기반** | **Figma**로 완성된 UX/UI 디자인을 JSP와 CSS로 구현하여 Design-to-View 역량 증명. |
| **Figma 링크** |  [[UI](https://www.figma.com/proto/oY469eP1SagPnU0ciUV37g/Untitled?node-id=4-3&t=5aqz34s41EG1cmJv-1)] |
| **강조 역량** | Spring MVC의 깊은 이해, Spring Security를 활용한 역할 기반 권한 관리, 복잡한 예약 트랜잭션 처리. |

# 💻 기술 스택 (Tech Stack)
| 분야 | 핵심 기술 | 사용 목적 |
| :--- | :--- | :--- |
| **백엔드/서버** | **Spring Boot** | **복잡한 매칭 알고리즘 및 예약 트랜잭션** 관리, Spring MVC의 Controller/Service 역할. |
| **뷰 (View)** | **JSP (JavaServer Pages)** | 동적인 HTML 생성 및 **Figma 디자인을 구현**하는 View 계층 역할 (JSTL 활용). |
| **보안** | **Spring Security** | 멘토/멘티/관리자 **역할 기반의 인증 및 권한 관리**. |
| **데이터베이스** | **MariaDB** | 멘토/멘티 정보, 예약 기록 등 관계형 데이터 관리. |
| **ORM** | **Spring Data JPA** | 데이터베이스 접근 및 객체-관계 매핑(ORM) 구현. |

# ⚙️ 핵심 구현 기능 표
| 구분 | 상세 기능 |
| :--- | :--- |
| **멘토-멘티 매칭** | 멘토의 전문 분야와 멘티의 요청사항을 분석하여 **적합도 기반 추천 목록** 구현. |
| **예약 시스템** | 멘토의 **가용 시간표**를 UI로 제공하고, **예약 중복을 방지**하는 트랜잭션 로직 구현. |
| **권한 관리** | Spring Security를 활용하여 역할(멘토/멘티/관리자)에 따라 접근 가능한 JSP 페이지를 구분하여 렌더링. |
| **피드백/평점** | 세션 종료 후 멘토와 멘티 간의 상호 피드백 및 평점을 기록하는 CRUD 기능. |

# 커밋 메시지

| Type | 목적 | 상세 설명 |
| :--- | :--- | :--- |
| **feat** | **새 기능 추가** | 새로운 기능(Feature)을 코드에 추가했을 때. |
| **fix** | **버그 수정** | 잘못된 코드를 고치거나 버그(Bug)를 수정했을 때. |
| **refactor** | **코드 구조 개선** | 기능 변경 없이 코드 구조를 개선(Refactoring)했을 때. |
| **docs** | **문서 수정** | 코드 외 문서(README, 주석 등)를 수정했을 때. |
| **chore** | **기타 설정 변경** | 빌드 환경, 라이브러리, 환경 설정 등 주요 로직 외 변경. |
| **style** | **스타일/포맷팅** | 코드 로직 변경 없이 스타일, 공백, 포맷팅만 변경했을 때. |
| **test** | **테스트 코드** | 테스트 코드를 추가하거나 기존 테스트를 수정했을 때. |
| **perf** | **성능 개선** | 코드 변경으로 성능(Performance)이 향상되었을 때. |
